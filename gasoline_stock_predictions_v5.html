The regime detector classifies the current market environment. Season is based on gasoline demand cycle (draw = spring drawdowns, peak = summer, build = fall restocking, shoulder = winter). Volatility ratio compares recent 8-week volatility to the trailing 52-week baseline (>1.5x = high vol). Turnaround flags when refinery utilization drops >3 pts in 4 weeks. Momentum measures whether stock changes are accelerating (+) or decelerating (-).
REGIME FEATURE VECTOR (what the meta-learner sees)
FeatureCurrent ValueDescription
These 7 values are appended to the 3 model predictions to form the meta-learner's 10-feature input vector.
MODEL BLEND WEIGHTS
How much each model's prediction contributes to the final blend. These multiply against ~240k stock values, so small coefficients produce large contributions. A negative weight means the meta-learner learned that model tends to overshoot and subtracts part of its prediction.
REGIME BIAS ADJUSTMENTS
Context-dependent corrections (in kbbl) that shift the prediction up or down based on current market regime. These multiply against 0/1 flags or small numbers, so the coefficients are large. Only active features (non-zero) contribute to the current forecast.
BLENDING WALKTHROUGH — Step by Step
For each forecast week, the meta-learner computes: prediction = intercept + c1 × S&D_Reg + c2 × S&D_Bal + c3 × AR + c4 × is_draw + ... + c10 × momentum. If the result falls outside a reasonable range, it falls back to the inverse-MAE weighted average.

FALLBACK: INVERSE-MAE WEIGHTS
When the meta-learner is inactive or its output fails the sanity check, the ensemble falls back to these weights (calculated from recent backtest MAE — lower error = higher weight, with exponential decay favoring recent weeks).
